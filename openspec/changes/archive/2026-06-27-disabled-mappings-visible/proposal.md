# 概述
当模型映射关联的厂商被禁用、或厂商模型被禁用时，模型映射详情页面不再隐藏这些映射，而是显示它们并标记为"强制禁用"状态。管理员可以看到并编辑/删除这些映射。

# 背景
当前模型详情页面（GET /api/v1/models/:id）在查询 ModelMapping 时，通过 `JOIN providers AND providers.enabled = ? true` 过滤掉了被禁用厂商的映射记录。这导致：
- 厂商被禁用后，映射在详情页面中完全不可见
- 厂商模型被禁用后，映射也不可见
- 管理员无法查看或管理这些"消失"的映射
- API Key 模型列表中映射统计（mapping_count）不准确

# 目标
- 所有 ModelMapping 记录在模型详情页面中都可见
- 被禁用的映射显示为灰色禁用开关（不可操作），鼠标悬停显示精确禁用原因（"厂商已禁用" / "厂商模型已禁用"）
- 手动禁用的映射行为不变（开关可操作）
- 聚合统计（mapping_count、min_context_window 等）行为不变，仍只反映可用映射
- API Key 模型列表中的映射统计包含所有厂商映射

# 非目标
- 不改变路由逻辑中排除禁用映射的行为
- 不改变添加映射对话框中禁用厂商/模型的行为（保持现状）
- 不改变批量删除对禁用的映射的处理
